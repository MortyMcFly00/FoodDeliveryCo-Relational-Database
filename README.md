# FoodDeliveryCo-Relational-Database

FoodDeliveryCo Relational Database is a SQL database project designed to model the operations of a food delivery business. The project includes related tables for customers, employees, vendors, inventory, menu items, carts, and orders, with primary and foreign key relationships, audit triggers, and sample data for testing and development.

## Project Overview

This project was created as a final assignment for a SQL course. Its purpose was to design and build a relational database that reflects how different parts of a food delivery business connect through structured data.

The database models business entities such as facilities, departments, employees, customers, vendors, menu items, stock, carts, and orders. It also includes sample data to demonstrate how the schema can be populated for testing and query development.

## Tech Used

- SQL
- MySQL

## Features

- Creates a relational database schema for a food delivery business
- Defines multiple connected tables for core business operations
- Uses primary keys and foreign keys to enforce table relationships
- Includes audit triggers for employee insert and delete activity
- Populates the database with sample operational data
- Supports structured business data across customers, vendors, inventory, employees, and orders
